# Supported Workflows

## Stronger supported workflow categories
- messaging-based interaction through Telegram
- local file and folder workflow support
- structured workflow setup and organization
- research and analysis support
- practical personal operations workflows

## More environment-dependent / variable workflows
- advanced browser automation
- account-sensitive live editing flows
- GUI-heavy desktop flows that depend on local state
- actions that require elevated permissions or fragile site interactions

## Why this matters
This helps separate repeatable/core flows from workflows that are more dependent on environment, permissions, or local UI conditions.
